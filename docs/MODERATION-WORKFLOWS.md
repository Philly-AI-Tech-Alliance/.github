## Daily Moderation Checklist

### Morning Check (10-15 minutes)
- [ ] Review new GitHub Discussions posts
- [ ] Check for new issues across repositories
- [ ] Scan Slack channels for overnight activity (when available)
- [ ] Review any reported content or flagged items
- [ ] Check moderator coordination channel for updates
- [ ] Review open moderation incidents log

### Ongoing Throughout Day (as needed)
- [ ] Respond to @moderator mentions
- [ ] Address reported Code of Conduct violations
- [ ] Welcome new community members
- [ ] Redirect off-topic discussions
- [ ] Lock resolved or unproductive threads
- [ ] Approve first-time contributors (if using pre-moderation)

### Evening Check (5-10 minutes)
- [ ] Final scan of Discussions and Slack
- [ ] Update moderation log if needed
- [ ] Note any items needing follow-up
- [ ] Hand off to next moderator if rotating coverage

### Weekly Tasks
- [ ] Review moderation metrics (number of actions, types of issues)
- [ ] Check in with other moderators on challenging cases
- [ ] Update moderation documentation if needed
- [ ] Report to Core Team on moderation activities

## Common Scenarios

### 1. Spam Posts

**Identifying Spam:**
- Generic messages with links to external sites
- Promotional content for unrelated products/services
- Duplicate posts across multiple threads
- Newly created accounts posting commercial content
- Messages with suspicious links or attachments

**Action Steps:**
1. **Verify**: Check poster's profile and history
2. **Delete**: Remove the spam content immediately
3. **Block**: Ban the user if clearly a spam account
4. **Report**: Flag to GitHub or platform if automated spam
5. **Document**: Log in moderation tracking sheet

**Response**: No need to respond to obvious spam. Delete and move on.

**For Borderline Cases** (might be legitimate but promotional):
- Leave a comment asking them to review community guidelines
- Give 24 hours to respond
- Remove if no response or if guidelines not followed

---

### 2. Off-Topic Discussions

**Examples:**
- Job postings in technical discussion threads
- Political debates in project channels
- Personal conversations clogging important threads
- Unrelated questions in focused topics

**Action Steps:**
1. **Assess**: Is it harmful or just misplaced?
2. **Redirect**: Gently point to appropriate channel/thread
3. **Move**: Transfer to correct location if possible
4. **Lock**: If thread has derailed significantly
5. **Clean**: Consider hiding off-topic comments if excessive

**Response Template:** See [Redirect to Appropriate Channel](#redirect-to-appropriate-channel)

**When to Act:**
- Immediately if disrupting important discussions
- After 3-4 off-topic messages in succession
- If original poster or others express frustration

---

### 3. Heated Debates

**Warning Signs:**
- Personal attacks ("you're stupid," "you don't know what you're talking about")
- Escalating language and tone
- Multiple back-and-forth argumentative posts
- Participants refusing to disengage
- Other community members expressing discomfort

**Action Steps:**
1. **Monitor**: Watch for Code of Conduct violations
2. **Cool Down**: Post a cooling-off message
3. **Private Messages**: Reach out to participants privately
4. **Lock Thread**: Temporarily lock if discussion is unproductive
5. **Separate**: Move technical disagreement to new focused thread if valid points are being lost

**Response Template:** See [Cooling Off Period](#cooling-off-period)

**Important:**
- Distinguish between passionate technical debate (good) and personal attacks (bad)
- Allow disagreement but require respect
- Don't take sides on technical issues unless Code of Conduct is violated
- Escalate if participants don't respond to moderation

---

### 4. Code of Conduct Violations

**Types of Violations:**
- **Minor**: Unintentional microaggressions, borderline language, first-time minor infractions
- **Moderate**: Repeated minor violations, deliberate disrespect, harassment, discriminatory language
- **Severe**: Threats, doxxing, sustained harassment, severe discriminatory behavior

**Response by Severity:**

#### Minor Violations
1. **Public or Private Reminder**: Point out the issue
2. **Explain**: Why it's problematic and what to do instead
3. **Give Grace**: Assume good intent on first occurrence
4. **Document**: Log the incident
5. **Watch**: Monitor for pattern

**Response Template:** See [Friendly Code of Conduct Reminder](#friendly-code-of-conduct-reminder)

#### Moderate Violations
1. **Remove Content**: Hide or delete the offending content
2. **Private Warning**: Send formal warning via DM
3. **Document**: Create detailed log entry
4. **Inform**: Notify other moderators and Core Team
5. **Temporary Action**: Consider temporary mute or ban (24-48 hours)
6. **Follow Up**: Check in after cooling period

**Response Template:** See [Formal Warning](#formal-warning)

#### Severe Violations
1. **Immediate Action**: Remove content and ban user immediately
2. **Protect Victim**: Reach out to affected person, offer support
3. **Escalate**: Notify Core Team immediately
4. **Document**: Create comprehensive incident report
5. **Review**: Core Team review within 24 hours
6. **Communicate**: Inform community if appropriate (protect privacy)

**Response Template:** See [Severe Violation Response](#severe-violation-response)

---

### 5. Welcoming New Members

**Opportunities to Welcome:**
- First-time post in Discussions
- First PR or issue opened
- Introduction posts
- First-time attendee at events

**Action Steps:**
1. **Respond Quickly**: Within 24 hours if possible
2. **Be Warm**: Use friendly, encouraging tone
3. **Provide Guidance**: Point to relevant resources
4. **Offer Help**: Let them know you're available
5. **Tag Others**: Connect them with relevant people if appropriate

**Response Template:** See [Welcome New Member](#welcome-new-member)

## Response Templates

### Redirect to Appropriate Channel

```
Hi @username,

Thanks for your post! This topic would be better suited for [APPROPRIATE CHANNEL/THREAD].
Could you please repost there? You'll likely get better engagement and responses.

Here's the link: [LINK]

If you need help finding the right place, feel free to ask!
```

---

### Cooling Off Period

```
Hi everyone,

I'm stepping in as a moderator to ask that we take a step back from this discussion.
While it's great to see passionate engagement, this thread has become heated and is
straying from productive discussion.

Let's take a 24-hour cooling-off period. If you'd like to continue the technical
discussion after that, please focus on the ideas and proposals rather than personal
disagreements.

Remember our Code of Conduct: we discuss ideas, not people.

If anyone would like to discuss this moderation action, feel free to DM me directly.

I'm locking this thread for 24 hours. Thank you for understanding.
```

---

### Friendly Code of Conduct Reminder

```
Hi @username,

I wanted to reach out about [SPECIFIC COMMENT/BEHAVIOR]. While I'm sure it wasn't
intentional, this comment could be interpreted as [EXPLAIN ISSUE - e.g., dismissive
of others' contributions, using language that may make some community members
uncomfortable].

Our Code of Conduct asks that we [SPECIFIC GUIDELINE].

I'm not taking any action here, just wanted to bring it to your attention. I
appreciate your understanding and your contributions to the community!

If you'd like to discuss this further, I'm happy to chat.

Best,
[YOUR NAME]
Moderator
```

---

### Formal Warning

```
Hi @username,

I'm reaching out as a moderator regarding [SPECIFIC INCIDENT] on [DATE].

This behavior violates our Code of Conduct, specifically: [CITE SPECIFIC SECTION].

[EXPLAIN WHY IT'S PROBLEMATIC]

I've removed the content in question. This is a formal warning. If similar behavior
occurs again, we may need to take further action including temporary suspension or
permanent ban.

We value your contributions to the community, and I hope we can move forward
positively. Please review our Code of Conduct: [LINK]

If you'd like to discuss this warning or have questions, please reply to this message.

Best,
[YOUR NAME]
Moderator

---
This warning has been logged in our moderation records.
```

---

```
[PRIVATE MESSAGE TO USER]

Your account has been suspended from the Philly AI Tech Alliance community due to
[SPECIFIC SEVERE VIOLATION] on [DATE].

This behavior is unacceptable and violates our Code of Conduct.

If you believe this suspension is in error, you may appeal by contacting

[PUBLIC STATEMENT TO COMMUNITY - IF APPROPRIATE]

The moderation team has taken action on a serious Code of Conduct violation. We've
suspended an account and removed related content. We take these matters seriously
and want to ensure our community remains safe and welcoming for all.

If anyone was affected by this incident and would like support, please reach out to
moderators or core-team@philly-ai-tech-alliance.org.

```

## Escalation Process

### When to Escalate

Escalate to Core Team when:
- Code of Conduct violation is severe or unclear how to handle
- User disputes a moderation action
- Pattern of violations from same user
- Moderators disagree on appropriate action
- Legal concerns (threats, illegal activity, etc.)
- Need to coordinate larger community response
- Media or public attention on an incident
- You feel uncomfortable handling alone

### How to Escalate

1. **Immediate Needs**:
   - Post in private moderator channel: "Escalating incident, need Core Team ASAP"
   - Include link to incident and brief description
   - Tag Core Team members

2. **Non-Urgent**:
   - Create detailed incident report (see logging section)
   - Email core-team@philly-ai-tech-alliance.org
   - Subject: "Moderation Escalation: [BRIEF DESCRIPTION]"
   - Include all relevant context and your recommendation

3. **Formal Appeal**:
   - User appeals moderation decision
   - Forward appeal to Core Team
   - Core Team reviews within 3-5 business days
   - Core Team decision is final

### Escalation Response Times
- **Severe/Urgent**: Core Team responds within 4 hours
- **Moderate**: Core Team responds within 24 hours
- **Low Priority**: Core Team responds within 3 business days

## Moderator Coordination

### Communication Channels
- **Primary**: Private #moderators Slack channel (when available)
- **Backup**: moderators@philly-ai-tech-alliance.org email list
- **Urgent**: Direct phone/text contact list (shared privately)

### Coordination Practices

#### Coverage Rotation
- Designate primary moderator for each day/week
- Post rotation schedule in moderator channel
- Backup moderator available if primary is unavailable
- Handoff notes at shift changes

#### Consistency
- Discuss challenging cases with other moderators before acting
- Share responses in moderator channel for feedback
- Document decisions and reasoning
- Align on interpretation of policies

#### Moderator Meetings
- **Weekly sync**: 30 minutes, review active issues
- **Monthly review**: 1 hour, evaluate policies and processes
- **Ad-hoc**: As needed for urgent matters

#### Moderator Support
- Moderation can be emotionally taxing
- Check in with each other regularly
- Share the load—no one moderator handles everything
- Take breaks if dealing with difficult content
- Escalate if you're feeling overwhelmed
- Core Team provides support and backup

## Logging and Tracking

### Why We Log
- Track patterns of behavior
- Provide context for future moderation decisions
- Protect moderators with documentation
- Generate metrics for improvement
- Support escalations and appeals

### What to Log

#### For All Actions
- Date and time
- Platform (GitHub, Slack, etc.)
- Link to content
- User involved
- Type of issue
- Action taken
- Moderator who took action

#### For Warnings and Bans
- Full description of violation
- Relevant Code of Conduct section
- Previous warnings or incidents
- Communication sent to user
- Response from user (if any)

### Where to Log

**Moderation Log Spreadsheet** (private, moderator access only):
- Columns: Date, Time, Platform, User, Issue Type, Action, Moderator, Notes, Link
- Maintained in secure shared drive
- Only accessible to moderators and Core Team

**GitHub Issues** (private repository):
- For significant incidents requiring tracking
- Use template for consistency
- Link to public content (when appropriate)
- Tag with labels: `moderation`, `code-of-conduct`, severity level

### Log Retention
- Keep records for 2 years
- After 2 years, anonymize and archive for policy improvement
- User privacy must be maintained

## Moderator Quick Reference

### Severity Assessment

| **Severity** | **Examples** | **Action** | **Response Time** |
|------------|------------|----------|-----------------|
| **Low** | Off-topic post, minor formatting issues | Redirect, friendly reminder | Within 24 hours |
| **Minor** | Unintentional insensitive language, first offense | Public or private reminder | Within 12 hours |
| **Moderate** | Repeated violations, deliberate disrespect | Warning, content removal, possible temp ban | Within 4 hours |
| **Severe** | Harassment, threats, discrimination | Immediate ban, Core Team notification | Immediately |

### Action Authority

| **Action** | **Moderator** | **Core Team** |

| Redirect/remind | ✅ | ✅ |

| Lock thread | ✅ | ✅ |
| Informal warning | ✅ | ✅ |
| Formal warning | ✅ (log) | ✅ |
| Temporary ban (≤48h) | ✅ (notify Core) | ✅ |
| Temporary ban (>48h) | ❌ | ✅ |
| Permanent ban | ❌ | ✅ |

## Training and Resources

### New Moderator Onboarding
1. Read Code of Conduct and Moderation Policy thoroughly
2. Review this workflows document
3. Shadow experienced moderator for 2 weeks
4. Practice with scenario role-plays
5. Get access to moderation tools and logs
6. Meet Core Team liaison
7. Start with low-stakes tasks (welcoming, redirecting)

### Ongoing Training
- Quarterly scenario discussions
- Updates when policies change
- Share learnings from challenging cases
